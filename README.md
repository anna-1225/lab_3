## Разработка грамматики
Формально грамматика записывается в виде:

G[Z] = (Vn, Vt, P, Z)

Vn = { Z, <присваивание>, <условное-выражение>, <логическое-выражение>, <логическое-или>, <логическое-и>, <логическое-не>, <отношение>, <операнд>, <оператор-отношения> }

Vt = { a..z, A..Z, 0..9, =, ?, :, ;, >, <, !, &, |, (, ), if, else, or, and, not}

P:

1. Z -> <присваивание>
2. <присваивание> -> id '=' <условное-выражение> ';'
3. <условное-выражение> -> <операнд> 'if' <логическое-выражение> 'else' <операнд>
4. <логическое-выражение> -> <логическое-или>
5. <логическое-или> -> <логическое-и>
6. <логическое-или> -> <логическое-и> 'or' <логическое-или>
7. <логическое-или> -> <логическое-и> '||' <логическое-или>
8. <логическое-и> -> <not-выражение>
9. <логическое-и> -> <not-выражение> 'and' <логическое-и>
10. <логическое-и> -> <not-выражение> '&&' <логическое-и>
11. <not-выражение> -> <отношение>
12. <not-выражение> -> 'not' <not-выражение>
13. <not-выражение> -> '!' <not-выражение>
14. <отношение> -> <операнд>
15. <отношение> -> <операнд> <оператор-отношения> <операнд>
16. <операнд> -> id
17. <операнд> -> num
18. <операнд> -> '(' <логическое-выражение> ')'
19. <оператор-отношения> -> '>'
20. <оператор-отношения> -> '<'
21. <оператор-отношения> -> '>='
22. <оператор-отношения> -> '<='
23. <оператор-отношения> -> '=='
24. <оператор-отношения> -> '!='

### Расшифровка обозначений
```
id -> letter {letter | digit}
num -> digit {digit}
letter -> 'a'..'z' | 'A'..'Z'
digit -> '0'..'9'
```
## Классификация грамматики (по Хомскому)
Разработанная грамматика является контекстно-свободной, то есть относится к грамматикам 2 типа по классификации Хомского.

Это объясняется тем, что каждое правило вывода имеет вид:
```
A -> α
```
где:

* A — один нетерминальный символ;
* α — последовательность терминальных и нетерминальных символов.

Так как в левой части каждого правила стоит ровно один нетерминал, разработанная грамматика является контекстно-свободной и допускает построение синтаксического анализатора методом рекурсивного спуска.

## Метод анализа
Так как разработанная грамматика является контекстно-свободной, для синтаксического анализа используется метод рекурсивного спуска.

Основная идея метода рекурсивного спуска состоит в том, что каждому нетерминалу грамматики ставится в соответствие отдельная процедура синтаксического анализа. Эти процедуры вызываются в соответствии с правилами грамматики и последовательно распознают корректную цепочку лексем.

В программе реализованы следующие основные процедуры:
* Parse
* ParseConditionalExpression
* ParseLogicalExpression 
* ParseRelation
* ParseOperand
* ParseComparisonOp
* SkipErrorTokens
* HandleInvalidOperators

Главной процедурой является Parse, которая соответствует начальному символу Z и анализирует конструкцию вида:
```
идентификатор = операнд if логическое_выражение else операнд ;
```

### Диагностика и нейтрализация синтаксических ошибок
Для диагностики и нейтрализации синтаксических ошибок в программе реализован механизм восстановления разбора, основанный на идее метода Айронса.

Основная идея состоит в том, что при возникновении синтаксической ошибки анализатор не прекращает работу полностью, а:

* фиксирует ошибочный фрагмент;
* выполняет восстановление по ближайшим точкам синхронизации;
* продолжает разбор оставшегося текста.

В данной работе это реализовано следующим образом:
* При обнаружении ошибки вызывается метод AddError(), который сохраняет ошибочный фрагмент, позицию ошибки, длину фрагмента и сообщение об ошибке
* Метод ReadInvalidSequence() считывает некорректную последовательность символов до ближайшего допустимого символа
* Методы Expect() и ExpectWord() выполняют проверку ожидаемых символов и ключевых слов

В качестве точек синхронизации используются ключевые элементы конструкции:
* "=" - оператор присваивания

* "if" - ключевое слово условия

* "else" - ключевое слово альтернативы

* ";" - признак конца оператора

Такой подход позволяет:

* не останавливаться после первой ошибки;
* находить несколько ошибок за один запуск;
* сохранять устойчивость анализа даже при многострочном вводе.
